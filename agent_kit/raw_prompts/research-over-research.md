以下是四层分级模版
**L0**
- 这个方案怎么做得更好？
- 哪个 prompt / tool / API 组合效果更好？
- 哪个实现更快、更便宜、更准？
- 这个 bug / failure case 怎么修？
- 这个局部改动有没有涨点？
- 在当前任务和当前 metric 下，最好的参数是什么？
- 哪个 ablation 真正有用？
- 哪个样例失败了，为什么？
- 这轮实验有没有超过当前 baseline？
- 还能用什么小改动继续提分？

**L1**
- 我们现在解决的任务定义对吗？
- 当前 metric 真代表真实价值吗？
- 有没有 proxy metric 误导我们？
- baseline 是不是太弱了？
- 有没有更合理的 baseline 应该先建立？
- 这个任务是不是应该重拆？
- 离线评测和线上表现为什么不一致？
- 当前实验反馈周期为什么这么慢？
- 如何缩短一轮实验从想法到结论的时间？
- 哪些变量最值得优先研究？
- 什么指标应该是主指标，什么只是 guardrail？
- 当前研究流程里最大的瓶颈是什么？

**L2**
- 哪些研究方向值得投入资源？
- 哪些方向应该停止？
- 哪些 baseline 应该变成团队标准资产？
- 哪些实验平台和评测框架应该产品化？
- 人力、算力、时间怎么分配最有效？
- 哪些问题共用同一套 infra，应该合并建设？
- 哪些 failure pattern 值得单独成立研究主题？
- 怎么减少重复试错？
- 怎么让团队更稳定地产出强 baseline？
- 哪些研究工作该自动化？
- 如何建立 hypothesis prioritization 和 kill criteria？
- 我们的研究组合现在是否平衡？
- 研究效率本身该怎么衡量？

**L3**
- 我们为什么做这些研究？
- 哪些研究方向真正服务公司战略？
- 未来 1 到 3 年最该押哪几个能力？
- 我们真正的技术壁垒应该是什么？
- 哪些研究值得长期下注，哪些只是短期优化？
- 我们是要赢在模型、系统、数据、产品，还是平台？
- 研究组织应该怎么设计？
- 基础研究、应用研究、平台建设比例如何分配？
- 什么叫“研究成功”？
- 哪些方向虽然能涨分，但战略价值低？
- 哪些新范式值得提前布局？
- 这套研究投入最终会改变什么业务结果？
- 我们要成为什么样的组织，而不只是做什么实验？
