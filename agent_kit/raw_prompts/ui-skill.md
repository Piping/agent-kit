- 输入：分析本地文件 $1（以 HTML 里出现的全部区块为准）。
- 目标：从该模板抽象出可复用的「设计系统 + 组件库」，同时保持原模板的视觉风格并做系统化整理。
- 交付物：需要 A/B/C 都要——UI 设计方案 Markdown、Design tokens（CSS variables）、可落地的具体 CSS。
- 技术栈/落地：纯 HTML + CSS（不需要真实 CSS Modules 构建/哈希隔离；按模块化拆分/组织即可）。
- 命名规范：更工程化命名，采用 BEM；组件命名形如 ui-*、ui-*-__*、ui-*--*。
- “CSS 组合”偏好：以组件化为优先（通过组件类 + modifier 组合完成变体），而不是 utility 堆叠。
- 响应式：需要支持移动端，断点使用常见 Webflow 断点体系。
- 可访问性：需要包含（focus 状态、减少动效等）并在方案中覆盖。
- 约束：颜色必须与原模板一致；字体允许修改/替换。
